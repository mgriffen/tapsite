Section I — Mission Brief

A persistent, authenticated Chromium
under Claude's direct control.

cbrowser is a local MCP server that gives Claude a real browser with persistent sessions. Navigate to any web dashboard—including those behind MFA—and deploy 37 extraction tools to pull design tokens, API schemas, accessibility audits, performance metrics, and full visual reports. No credentials ever enter the chat.

Built for modernizing legacy admin UIs when you have browser access but zero codebase access.

How sessions work: You log into the target dashboard once in a headed browser window, complete any MFA challenges yourself, and cbrowser saves the session state to a local Playwright persistent context. Every subsequent tool call reuses that authenticated session automatically. Session cookies survive across restarts.

Section III — Operational Workflow

Three directives.

1
Authenticate once

Open a headed browser. Log in manually. Complete MFA yourself. The session is saved to a persistent Chromium profile on your machine. You won't log in again until the session expires.

cbrowser_login_manual
2
Navigate & extract

Ask Claude to navigate to any page within the authenticated session. Run any combination of the 37 tools—inspect, screenshot, extract design tokens, capture network traffic, audit accessibility. No credentials in the chat, ever.

cbrowser_navigate + any tool
3
Export artifacts

Generate a full design system report with visual HTML, W3C design token JSON, copy-paste CSS variables, Markdown documentation, CSV table exports, and page screenshots—all in one command.

cbrowser_export_design_report
^ that's literally the whole workflow. three commands.
Section IV — Field Notes

Why this works.

Asset: Session Persistence
MFA-safe session reuse

Playwright persistent context keeps cookies and localStorage alive across runs. Complete MFA once, reuse the session for days. Credentials never touch Anthropic's servers.

Asset: Design Intelligence
Full design system in one pass

Colors, fonts, spacing, CSS vars, components, breakpoints—extracted to W3C design token JSON and copy-paste CSS. Ready for a redesign brief in minutes, not weeks.

Asset: Network Recon
API schema without source access

Intercepts network requests to infer API structures. Detects the frontend stack from loaded scripts and response headers. Works when you have zero codebase access.

Asset: Operational Security
Entirely local execution

No third-party browser service. No data leaving your network. Just Node.js, Playwright, and a persistent Chromium profile stored in your project folder.

Section V — Deployment

Three commands. Then it's live.

# install dependencies
npm install
npx playwright install chromium
# add to Claude config (~/.claude/.mcp.json)
{
  "mcpServers": {
    "cbrowser": {
      "command": "node",
      "args": ["/path/to/cbrowser/src/mcp-server.js"]
    }
  }
}
# restart Claude Code → 37 tools appear

That's the entire deployment. No cloud services. No API keys beyond what you already have. No Docker, no database, no build step. Install, configure, restart.
